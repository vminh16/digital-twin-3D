# Phase 4.5 30k Dry Run Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a locked HCM0181 30k internal-holdout dry run with one rolling recovery checkpoint and bounded initialization/final evaluation artifacts.

**Architecture:** Extend the existing trainer with an opt-in fixed checkpoint path, and extend the existing qualification entry point rather than creating a second training engine. A small Bash runner locks the operational command and resume path. Existing Phase 4.4 and ordinary training behavior remain unchanged.

**Tech Stack:** Python 3.12, PyTorch, gsplat 1.4.0, NumPy, Pillow, Bash, pytest.

## Global Constraints

- Full-length qualification is exactly HCM0181, factor 1, seed 0, 30,000 steps, internal holdout, cached images, pinned transfer, and `grow_grad2d=0.0002`.
- Resume is allowed only from the same run's `checkpoints/recovery.pt`.
- Checkpoints remain atomic and only one full optimizer checkpoint is retained.
- Validation runs only at initialization and step 30,000; only final validation writes images.
- Public `test/images` is never read in Phase 4.5.
- A clean Git commit is required and recorded.
- Work directly on `main` as explicitly requested by the user.
